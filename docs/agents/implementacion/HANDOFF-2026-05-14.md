## Decisiones importantes tomadas hoy (no las revisites)

1. **`AgentHandoff.from_yaml` es la fuente única de validación.** El helper `_validate_handoff_text` solo formatea errores y agrega `expected_agent` opcional. Si necesitás cambiar reglas de validación, tocá `cortex/handoff.py` (NO el helper).

2. **`_verify_session_claims_text` usa subprocess directo a `git diff`.** No depende de `cortex.doc_verifier` para evitar acoplar el MCP server al verificador de docs. Si Plan 07 quiere unificar, hay que considerar el costo de la dependencia.

3. **El bucket `contradicted` está declarado pero vacío.** El heurístico solo distingue verified (≥2 tokens del claim aparecen en el diff) vs asserted. Detectar contradicciones requiere heurística de negación que no está en alcance de Plan 02. Plan 07 puede agregarla o dejarla como mejora futura.

4. **`MemoryEntry.confidence` es type-checked, no metadata dict.** El plan original sugería `metadata.get("confidence")` pero usé el campo `confidence` directo agregado en Plan 01 §6. Los wrappers de prompt (`RetrievalResult.to_prompt`, `EnrichedContext.to_prompt_format`) lo leen de ahí.

5. **Las 4 secciones nuevas de session note (Verified State / Unverified Claims / Blockers / Suggested Skills) solo se emiten si las listas son no-vacías.** Decisión para evitar ruido en sesiones normales que no son handoffs.
