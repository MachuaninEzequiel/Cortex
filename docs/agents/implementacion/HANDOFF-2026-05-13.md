## Decisiones importantes tomadas hoy (no las revisites)

1. **"Tripartita Refinada" ≠ "Ola 5".** Las olas son features pre-adopters (cerradas). Tripartita Refinada es feature post-adopters. Bumpea a **0.5.0** cuando se cierren los 7 planes.

2. **Schema `AgentHandoff` soporta 7 agent names** (5 generales + 2 Pi-only). Los Pi-only son `cortex-security-auditor` y `cortex-test-verifier`. Decisión tomada para evitar tener dos schemas duplicados. Si en el futuro Pi agrega más agents, hay que extender el Literal.

3. **CONTEXT.md es opcional.** Lo creé como template en `_create_vault_docs`. Si el adopter no lo edita, queda con la tabla vacía y los skills lo leen sin problemas (lo encuentran pero no encuentran términos). Si lo borra, los skills (post update) lo detectan y simplemente no lo cargan.

4. **`_meets_adr_criteria` es module-level, no método.** Decisión: el filtro es lógica pura sin estado, debería poder llamarse desde cualquier parte del codebase sin instanciar `DocGenerator`. Si en Plan 07 lo encontrás como método, alguien lo movió mal.

5. **El backend del Verification Gate vive en Plan 02 (MCP tool `cortex_verify_session_claims`).** Hoy solo el prompt canonical del documenter referencia el tool. **El tool no existe aún.** Si arrancás Plan 02 antes que cualquier IDE corra el documenter actualizado, todo bien. Si un IDE corre el documenter sin tener Plan 02 mergeado, el agente falla al invocar el tool inexistente. **Mergear Plan 01 + Plan 02 juntos antes de bumpear versión.**
